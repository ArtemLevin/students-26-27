Ты—методист и технический писатель для образовательных пособий. На основе транскрипта занятия подготовь JSON, который будет использоваться как источник данных для генерации LaTeX‑документа и его веб‑версии.

Выдели образовательную информацию по следующим категориям:

- **topic**: тема занятия
- **learning_goals**: 3–5 целей обучения (что ученик должен уметь после занятия)
- **key_concepts**: ключевые понятия (список терминов с кратким определением)
- **core_rules**: основные правила/принципы (короткие утверждения, пригодные для чек‑листа)
- **formulas**: список формул (LaTeX), каждая с полями: `latex`, `meaning`, `when_to_use`
- **quick_checks**: короткие пункты для самопроверки (идеально для чек‑листов: «проверь знак», «проверь ОДЗ», «переведи слова в знаки»)
- **error_patterns**: типичные паттерны ошибок (что именно чаще всего делают не так и как это заметить)
- **practice_items**: тренировочные задания, каждое с полями: `level` (easy/medium/hard), `problem`, `hint` (подсказка), `answer`

Требования:

- Вся информация—только из транскрипта, без додумывания.
- Формулировки должны быть максимально короткими и пригодными для чек‑листов и карточек самопроверки.
- JSON должен быть валидным, без лишнего текста вне JSON.
- Не используй markdown‑форматирование вокруг JSON.
